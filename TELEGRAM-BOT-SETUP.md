# Telegram-бот для заявок aotrom.art

Бот уже создан через официальный [@BotFather](https://t.me/BotFather): `@aotrombot`. Старый бот и личный Telegram продолжают работать независимо.

## Безопасная схема

1. Форма отправляет данные на `/api/inquiry`.
2. Серверная функция читает `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` из защищённых переменных Vercel.
3. Заявка приходит в чат владельца через Bot API.
4. Если серверная доставка недоступна, сайт открывает резервный черновик для `@aotrom0`.

Токен нельзя вставлять в код сайта, браузерный JavaScript, сообщения в публичных чатах или GitHub. В репозитории есть только названия переменных, но не их значения.

## Что сохранить в Vercel

- `TELEGRAM_BOT_TOKEN` — актуальный токен, полученный у @BotFather.
- `TELEGRAM_CHAT_ID` — числовой ID чата получателя заявок.

Обе переменные нужны для Production, Preview и Development. После изменения переменных необходимо выполнить новый deployment.

## Токен был показан в переписке

После первичной проверки нужно открыть @BotFather, отправить `/revoke`, выбрать `@aotrombot`, получить новый токен и сразу заменить значение `TELEGRAM_BOT_TOKEN` в Vercel. Старый токен после `/revoke` перестанет работать.

## Как создать ещё одного бота при необходимости

В @BotFather отправьте `/newbot`, задайте видимое имя и уникальный username, заканчивающийся на `bot`. Затем откройте нового бота и отправьте `/start`. Новый токен храните только как серверный секрет.
